# Tandayapa Soundscape

Passive-acoustic **recorder-spacing study** in the Tandayapa cloud forest, Ecuador, with
a full analysis + verification pipeline: acoustic indices, automated species ID
(BirdNET for birds, a custom classifier for frogs, BirdNET's insect classes), a
pairwise distance-decay analysis, and an interactive **web app** to verify every
detection by ear + spectrogram.

> **This repo holds the *code, docs, and reference lists* only.** Raw audio, trained
> models, and derived outputs are excluded (see [What's *not* in the repo](#whats-not-in-the-repo)).
> The main reason to clone it on a stronger machine is to **train high-precision
> frog and insect classifiers** for Tandayapa using Google's **Perch** model —
> see [Training new models](#training-new-models-the-main-goal).

---

## The study

**Question.** How does the similarity of sound recorded by two AudioMoths change with
the distance between them, and is that distance–similarity relationship different in
forest than in pasture? Measured with acoustic indices (ACI, BI, ADI), direct spectral
similarity, and species-community overlap.

**Hypothesis.** Two recorders record less similar sound as the distance between them
grows (distance decay), and similarity drops over **shorter distances in forest** than
in pasture, because forest vegetation/terrain scatter and absorb sound so each recorder
"hears" a smaller area.

**Design.** 3 recorders per habitat (forest, pasture), placed in a line ~20 m inside the
forest edge / in open pasture, at a fixed spacing that changes each deployment day:

| Day  | Spacing | Date (2026) | Recorders |
|------|---------|-------------|-----------|
| day1 | 15 m    | Jun 2       | 3 forest + 3 pasture |
| day2 | 30 m    | Jun 3       | 3 forest + 2 pasture (1 failed) |
| (day3) | 60 m  | Jun 4       | audio collected; not yet configured |

Within-habitat recorder **pairs** are the unit of the distance analysis; 10-/2-min clips
and hours are temporal subsamples summarised to the hour, never independent replicates.
This is a **methodological pilot** (small n) — honest about its limits.

### ⚠️ Two field gotchas baked into the config
- **Clock offset.** Devices `F1503_A08` and `F3003_A07` were set to **UTC (5 h fast)**.
  The pipeline corrects them with `offset_h = -5` in `DEVICES` (see `tandayapa_common.R`).
  Always check a new deployment for this — the filename time is the device clock.
- **AudioMoth format.** 48 kHz, mono, 16-bit; 2-min clips every 20 min (3 clips/hour).

All field metadata — devices, per-day analysis windows, spacings, clock offsets — lives in
**`tandayapa_common.R`** (single source of truth, sourced by every script).

## Setup on a new machine

1. **Paths.** Scripts currently hard-code `d:/Audiomoths`. After cloning, update:
   - `ROOT_DIR` at the top of `tandayapa_common.R`,
   - the `source(file.path("d:/Audiomoths", ...))` lines at the top of each runner,
   - `FROG_DIR` in `run_frogs_day1.R` (path to the frog model folder).
   (A future cleanup would read these from one env var / `here::here()`.)

2. **Audio.** Put each AudioMoth's WAV folder back at the repo root using the exact names
   in `DEVICES` (e.g. `F1501_A05/20260602_104000.WAV`).

3. **R** (≥ 4.4). Install packages:
   ```r
   install.packages(c("soundecology","tuneR","seewave","lubridate","dplyr","tidyr",
     "stringr","purrr","readr","ggplot2","tibble","jsonlite","future","furrr",
     "lme4","lmerTest","rmarkdown",
     "birdnetR","reticulate"))   # birdnetR auto-provisions a Python+TF backend on first model load
   ```

4. **Web app** (Node ≥ 18):
   ```bash
   cd webapp && npm install
   ```

## Running the pipeline (per day)

`DAY <- "day1"` is set at the top of each runner; change it (and re-run) for other days.

```bash
# 1. Birds: score every clip with BirdNET  (~35 min/day, cached)
Rscript run_birdnet_day1.R
# 2. Range filter: the species BirdNET expects at Tandayapa now (cuts out-of-range FPs)
Rscript birdnet_location_filter.R
# 3. Frogs: score every clip with the custom Ecuador frog model (needs models/, ~25 min/day)
Rscript run_frogs_day1.R
# 4. Export merged bird+frog+insect detections to JSON for the web app
Rscript export_web_data.R
# 5. Analysis report (acoustic indices -> pairwise distance decay -> figures)
Rscript -e 'rmarkdown::render("pilot_day1_acoustic_analysis.Rmd")'
```

### The verification web app
A React app to play each detection (original WAV streamed over the LAN) with a live
spectrogram + the detection region, filterable by **taxon (🐦 / 🐸 / 🦗), day, habitat,
recorder, time, confidence**.
```bash
cd webapp && npm run build        # outputs webapp/dist/
# then serve the REPO ROOT (so /outputs/web and the WAVs resolve) e.g.:
python -m http.server 8000        # open http://<this-pc>:8000/webapp/dist/
```
`serve_webapp.bat` does this for you. Data comes from `outputs/web/*.json`
(`export_web_data.R`); audio is served as `/<folder>/<file>.WAV` from the repo root.

## The models (and why they need improving)

- **Birds — BirdNET v2.4** via `birdnetR`. Good. We apply BirdNET's **location/time range
  filter** so only species plausible at Tandayapa are kept.
- **Frogs — a custom BirdNET classifier** (`models/frog_classifier_ec/`, **not in repo**)
  covering **122 Ecuadorian amphibians** (lots of *Pristimantis*, glass frogs, *Gastrotheca*…;
  full list in `reference/frog_classifier_species_ecuador.txt`). It correctly finds the
  forest *Pristimantis* night chorus, **but it is closed-set**: every 3-s window is forced
  into some frog class, so non-frog sounds become false IDs (e.g. *Pipa pipa* on noise).
  Only a handful of species verified as reliable.
- **Insects — BirdNET's Orthoptera/Cicadidae classes**, relabelled to **genus** (flagged
  `~`, e.g. `~ Neoconocephalus (katydid)`). The species are temperate/North-American, so
  treat **family** (katydid/cricket/cicada) as the only safe level and genus as approximate.

> Bottom line: the frog model has a **precision/open-set** problem, and there is **no
> good off-the-shelf model** for Tandayapa frogs or Neotropical insects. That is what the
> next step fixes.

## Training new models (the main goal)

Build **high-precision, local** frog and insect detectors with Google DeepMind's
**Perch 2.0** + the **`perch-hoplite`** agile-modeling workflow. The key advantage over the
current closed-set frog model: you label **negative** examples (wind, rain, birds, the
noises that trigger false frogs), so false positives largely disappear, and you only build
detectors for species/sounds **actually present** at Tandayapa.

**Why a powerful PC:** embedding many hours of audio and (optionally) fine-tuning is much
faster with an NVIDIA GPU.

### Workflow
1. Install Perch tooling (Python):
   ```bash
   pip install git+https://github.com/google-research/perch-hoplite.git
   # system deps: libsndfile, ffmpeg
   ```
   Repos/docs: <https://github.com/google-research/perch-hoplite> ·
   <https://github.com/google-research/perch> (see `agile_modeling.ipynb`) ·
   Perch 2.0 paper: <https://arxiv.org/abs/2508.04665>
2. **Embed** all Tandayapa recordings into the Perch embedding DB (one-time).
3. **Gather reference calls** for each target species (a few examples each) from
   xeno-canto / iNaturalist / AmphibiaWeb — used as the search seeds.
4. **Agile-label:** vector-search from each seed, mark results **relevant / irrelevant**
   (the verification web app in this repo is ideal for the by-ear/spectrogram check), and
   include negatives.
5. **Train** a small linear classifier on the embeddings (often < 1 h per signal), export,
   and run it over the recordings like BirdNET. Feed results back through
   `export_web_data.R` → the web app, same as the existing taxa.

### Target species — "reported at Tandayapa and surrounding areas"
Compile the local list from these (Tandayapa Valley, ~0.0° S, 78.68° W, ~1700–2200 m,
Chocó-Andes cloud forest):

- **Frogs**
  - Start from `reference/frog_classifier_species_ecuador.txt` (122 spp the current model covers).
  - **BIOWEB Ecuador – Anfibios del Ecuador** (PUCE): <https://bioweb.bio/faunaweb/amphibiaweb>
  - **AmphibiaWeb**: <https://amphibiaweb.org>
  - **iNaturalist** — filter Amphibia to the Tandayapa / Mindo / Chocó-Andino area:
    <https://www.inaturalist.org>
  - Likely key genera here: *Pristimantis* (many), *Hyloscirtus*, *Gastrotheca*,
    glass frogs (*Hyalinobatrachium, Nymphargus, Sachatamia, Espadarana*), *Hyloxalus*.
- **Insects** (Orthoptera: katydids/crickets; Cicadidae)
  - **iNaturalist** Orthoptera/Cicadidae for the same area (research-grade, with audio).
  - **Orthoptera Species File**: <http://orthoptera.speciesfile.org>
  - **xeno-canto** (has a grasshoppers/insects section): <https://xeno-canto.org>
  - Reference dataset for method/baselines: **InsectSet459**
    (<https://zenodo.org/records/14056458>, paper <https://arxiv.org/abs/2503.15074>).

Save your final per-taxon target lists in `reference/` (e.g. `frogs_tandayapa.txt`,
`insects_tandayapa.txt`) so training and the range filters can use them.
